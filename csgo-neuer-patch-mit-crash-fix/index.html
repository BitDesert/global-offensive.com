CS:GO Neuer Patch mit Crash-Fix

von ESPORTS NETWORK · 11. Februar 2013

Ab sofort steht ein neuer Patch für den Multiplayer-Shooter Counter-Strike: Global Offensive via Steam zum Download bereit

Nachdem es seit dem letzten Counter-Strike Global Offensive Update enorme Probleme mit der Map de_dust2_se gab wurde nun ein Crash-Fix released. Problem war dass die Map nicht mehr auf dem Server gespielt werden konnte dar sie ein Teil des integrierten Map Workshops ist. Versuchten also Spieler auf dem Server diese Map zu spielen kam beim Clienten ein Fehler in Verbindung mit der Map de_dust2_se. Dieser Fehler wurde nun unter anderem gefixt. Offiziell sagt Steam: „Das Update behebt unter anderem einen Crash-Bug, der beim Dedicated Server auftreten konnte.“

Des Weiteren hat das Team auf der Map »cs_assault« neue Spawn-Punkte eingeführt und weitere kleinere Fehler beseitigt. Das Update steht ab sofort via Steam zum Download bereit.

 

Folgende Patch Notes durch Steam wurden veröffentlicht:

Patch-Notes für Counter-Strike: Global Offensive

  • Added ‚workshop_start_map‘ command line param to specify which map in a workshop collection for the server to start on. Takes the desired map’s file id.
  • Password-protected community servers are no longer available for community matchmaking.
  • Map name is now correctly reported to master server.
  • Added spawn points in cs_assault.
  • Convars now get reset to defaults before server.cfg is executed.
  • Fixed a dedicated server crash after player disconnection.
  • Fixed respawn pop-up showing when you could not respawn in some cases.
  • Fixed bug in respawning if team spawn point counts were asymmetrical.
  • Fixed prop_dynamic_glow’s not getting a final network update.
  • Fixed webapi authkey behavior on Linux dedicated servers.
  • Fixed layout of player details screen.

 

Im Zuge dieses Updates wurden alle _se Maps, welche wir für Wettkämpfe nutzen von der Serverstandardinstallation entfernt und in einen Workshop geschoben.
Dieses wird zwangsläufig zu Problemen führen. Wenn ihr eine _se Map nutzen wollt, benutzt folgenden Befehl als Server Administrator:

„rcon host_workshop_map WorkshopID“

Alle derzeit verfügbaren _se-Maps des Mappools:
• „rcon host_workshop_map 125488374“ – zum Laden von dust2_se
• „rcon host_workshop_map 125499116“ – zum Laden von inferno_se
• „rcon host_workshop_map 125498231“ – zum Laden von train_se
• „rcon host_workshop_map 125498553“ – zum Laden von nuke_se
• „rcon host_workshop_map 125498851“ – zum Laden von dust_se
• „rcon host_workshop_map 125498692“ – zum Laden von aztec_se

Nach Ausführung des Befehl 3-5 Minuten warten und dann auf die Map wechseln.

Hinweis für Serverbesitzer: Wenn ihr in eurer Mapgroup als Startmap eine _se Map eingetragen habt, wird der Server nicht starten!

Wir gehen davon aus, dass die Möglichkeit Maps aus dem Workshop zu laden bald einfacher sein wird. Bis dahin bieten wir aber auch die Möglichkeit an, die standard Maps (ohne _se) zu spielen.
